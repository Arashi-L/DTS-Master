**產品簡介**

DTS Master 是一款專為企業級環境設計的磁碟安全擦除工具。在數位轉型浪潮下，資料資產已成為企業的核心價值，然而退役儲存媒體上的殘留資料可能導致嚴重資訊外洩。傳統刪除作業僅移除檔案索引，實際資料仍可透過進階復原技術還原。DTS Master 採用多種業界公認的資料銷毀標準，透過底層磁區直接存取技術，確保實體媒體上的每一位元資料被徹底覆寫，滿足 ISO/IEC 27001、NIST 800-88、GDPR 等合規要求。

本工具支援 HDD、SSD、NVMe 等各類儲存裝置，提供多種覆寫演算法及硬體級 ATA 安全擦除指令，並具備完整的驗證與審計功能，協助組織建立從採購到報廢的閉環資料安全生命週期。

**系統需求**

·        **作業系統**：Windows 10 64位元、Windows 11 或更高版本伺服器系統（Windows Server 2016+）

·        **硬體**：x64 架構處理器，建議 4GB 以上記憶體（視擦除裝置數量而定）

·        **磁碟介面**：支援 IDE、SATA、SCSI、SAS、NVMe、USB 等儲存裝置

·        **權限**：必須以**系統管理員**身分執行，才能取得磁碟底層存取權限

·        **其他**：.NET Framework 4.8 或更新版本（依賴 Windows API）

---

**安裝與部署**

DTS Master 為免安裝命令列工具，僅需將執行檔 `dtsmaster.exe` 放置於本機任意目錄，並確保以管理員權限開啟命令提示字元或 PowerShell 執行。

**部署方式**

·        **本機使用**：直接複製執行檔至目標主機。

·        **大規模部署**：可搭配遠端管理工具或批次腳本進行多主機操作。

·        **整合 SIEM**：工具執行記錄可匯出為標準文字檔，供第三方審計系統匯入。

**快速入門**

以下範例展示如何使用 DTS Master 進行基本擦除。

1.     **檢視磁碟資訊**  
執行命令：  
`dtsmaster.exe
\\.\PhysicalDrive0 --info`

輸出範例：

`Disk Information:`

`Device Path: \\.\PhysicalDrive0`

`Total Size: 256.06 GB`

`Write Permission: Yes`

`Estimated Erase Time:`

`  -
Zero Fill: 0h 17m 5s`

`  -
DoD Short: 0h 51m 15s`

`  -
DoD Long: 1h 59m 35s`

`  -
Gutmann: 9h 58m 10s`

2.     **執行零填滿擦除（單次覆寫）**  
`dtsmaster.exe
\\.\PhysicalDrive0 --zero --verbose `
系統會提示確認，輸入 YES 後開始擦除，並即時顯示進度。

3.     **執行 DoD 5220.22-M 短版（3次覆寫）**  
`dtsmaster.exe
\\.\PhysicalDrive0 --dod-short`

4.     **執行 ATA 安全擦除（若硬碟支援）**  
`dtsmaster.exe
\\.\PhysicalDrive0 --ata`

5.     **僅驗證磁碟是否已填零**  
`dtsmaster.exe
\\.\PhysicalDrive0 --verify`

**注意**：`\\.\PhysicalDrive0` 代表 Windows 中的實體磁碟編號，可透過磁碟管理工具確認正確索引。

**功能詳解**

**5.1** **磁碟資訊查詢**

使用 `--info` 參數可顯示目標磁碟的詳細資訊，包括：

·        裝置路徑

·        總容量（自動轉換為 B/KB/MB/GB/TB）

·        寫入權限狀態

·        依據磁碟容量估算各擦除方法所需時間

此功能可幫助操作者確認目標裝置，避免誤操作。

**5.2** **資料擦除方法**

DTS Master 支援多種業界標準資料銷毀演算法，所有寫入操作均透過底層磁區直接存取，確保跳過檔案系統快取。

| **方法** | **命令列選項** | **說明** | **符合標準** |
| --- | --- | --- | --- |
| 零填滿 | `--zero` | 單次全磁區寫入 0x00 | 基礎清除 |
| 隨機填滿 | `--random` | 單次全磁區寫入隨機資料 | 基礎清除 |
| DoD 短版 | `--dod-short` | 三次覆寫：0x00 → 0xFF → 隨機資料 | DoD 5220.22-M |
| DoD 長版 | `--dod-long` | 七次覆寫，包含多種模式與驗證 | DoD 5220.22-M（最高機密級） |
| Gutmann 演算法 | `--gutmann` | 35 次模式化覆寫，針對磁記錄特性最佳化 | Gutmann 標準 |

每種方法執行完畢後，工具會自動執行驗證程序（零值檢查），確保資料已成功覆寫。

**5.3 ATA** **安全擦除**

對於支援 ATA 指令集的硬碟（含 SSD、NVMe），DTS Master 可發送硬體級安全擦除指令。此操作由硬碟控制器內部執行，通常比軟體覆寫更快，且能重置 NAND 區塊（對 SSD 尤為重要）。

使用方式：  
`dtsmaster.exe
\\.\PhysicalDrive0 --ata`

若硬碟不支援 ATA 安全擦除，工具將自動降階使用軟體覆寫方法（預設為零填滿）。執行過程中請勿關閉電源。

**5.4** **驗證機制**

·        **寫入後驗證**：每完成一次覆寫，可選用` --verify `參數單獨檢查磁碟是否已全部為零。

·        **自動驗證**：執行軟體覆寫方法後，工具會自動進行驗證，確保資料銷毀完整。

·        **CRC** **校驗**（內部實作）：每次寫入操作均檢查寫入位元組數，確保資料完整寫入。

**進階技術特性**

**7.1** **底層 I/O 存取架構**

DTS Master 透過 Windows 裝置路徑（`\\.\PhysicalDriveN`）直接開啟磁碟，並使用 WriteFile / ReadFile 搭配 `FILE_FLAG_WRITE_THROUGH` 旗標，確保資料繞過作業系統快取，直達實體媒體。

**7.2** **多執行緒覆寫引擎**

工具採用「生產者-消費者」模型：

·        專用執行緒池產生隨機資料或模式資料。

·        主執行緒負責 DMA 寫入，減少 CPU 占用。

·        寫入緩衝區大小可動態調整（預設 1MB），以適應不同裝置效能。

**7.3** **完整性驗證系統**

·        **校驗和驗證**：寫入時記錄寫入長度，確保資料完整。

·        **統計隨機性測試**（內部）：可選用於檢測寫入資料的隨機性。

·        **磁軌模擬**：針對老式硬碟，可模擬磁頭偏移以處理邊緣磁區資料殘留（未來版本）。

**常見問題 (FAQ)**

**Q1****：執行時出現「存取被拒」錯誤？**  
A：請確認您已以「系統管理員」身分執行命令提示字元或 PowerShell。

**Q2****：如何確認要擦除的 PhysicalDrive 編號？**  
A：可使用 Windows 磁碟管理（diskmgmt.msc）或執行 wmic diskdrive list brief 檢視對應型號與容量。

**Q3****：ATA 安全擦除失敗怎麼辦？**  
A：若硬碟不支援 ATA 指令或指令執行失敗，工具會提示並可選擇繼續軟體覆寫。建議使用 `--dod-long` 等高強度方法。

**Q4****：擦除 SSD 是否會影響壽命？**  
A：SSD 支援 TRIM 與 ATA 安全擦除，可有效重置 NAND 區塊，對壽命影響較小。軟體覆寫會增加寫入量，建議優先使用硬體級安全擦除。

**Q5****：能否同時擦除多個磁碟？**  
A：目前版本為單一磁碟命令列工具。如需批次處理，可自行撰寫腳本呼叫多個執行個體（注意 I/O 資源競爭）。

**Q6****：Gutmann 35 次覆寫是否仍適用現代硬碟？**  
A：Gutmann 演算法最初針對老式 MFM/RLL 編碼硬碟設計。現代硬碟密度高，但基於合規需求，工具仍保留此方法。對於新式硬碟，DoD 短版或 ATA 安全擦除通常已足夠。
